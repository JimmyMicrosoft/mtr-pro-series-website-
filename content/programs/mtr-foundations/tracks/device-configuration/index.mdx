## Overview

Track 3 is where the installed hardware becomes a functioning conference room. The physical work is done — displays mounted, cables run, cameras positioned, rack dressed. Now the device needs to be configured: OOBE completed, resource account signed in, application settings applied, Windows OS hardened, and the room tested against the acceptance checklist before the client signs off.

This track covers both MTR Windows and MTR Android configuration paths, the OEM-specific differences between major Android platforms, and the XML configuration system that underpins advanced MTR Windows configuration and fleet management.

---

## What you will learn

By completing this track, you will be able to:

- Complete the MTR Windows OOBE from first power-on through the Teams application welcome screen, troubleshooting common first-boot failures
- Sign an MTR device into a resource account, verify sign-in success in both the application and Teams Admin Center, and diagnose sign-in failures using error messages and application logs
- Configure all major MTR application settings — display, audio, meetings, peripherals — for a given room scenario
- Set Windows OS power, update, and network settings to ensure reliable MTR operation
- Execute a complete post-installation acceptance test and produce a signed acceptance report
- Complete the OOBE process on Android MTR devices, understanding the key differences from the Windows path
- Navigate OEM-specific admin interfaces and cloud management portals for Poly, Yealink, Logitech, Neat, and DTEN platforms
- Write, validate, and deploy SkypeSettings.xml configuration files for common MTR deployment scenarios using all four deployment methods
