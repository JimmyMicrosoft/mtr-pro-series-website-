## Overview

MTR Android is the second major platform in the Teams Rooms ecosystem, and it is distinct from MTR Windows in almost every practical sense. The hardware is OEM-integrated rather than compute-plus-peripherals. The OOBE is OEM-specific rather than Windows-based. The admin access is through a web interface or OEM app rather than Windows Settings. The management tools include an OEM cloud portal alongside Teams Admin Center.

This module covers the Android MTR commissioning path from first boot through OEM management portal setup, and builds the platform knowledge needed to select the right Android OEM for a given project and set accurate expectations with clients about feature parity.

---

## What you will learn

By completing this module, you will be able to:

- Complete the Android MTR OOBE from first power-on, correctly handling firmware updates and account sign-in
- Identify the five major MTR Android OEMs — Poly, Yealink, Logitech, Neat, and DTEN — and their target use cases, management portals, and key differentiators
- Access the local web interface and OEM cloud portal for major Android platforms and perform common post-installation configuration tasks
- Evaluate whether MTR Android meets a client's feature requirements and communicate any feature gaps accurately
