## Overview

The Android MTR OOBE is more OEM-centric and less standardized than the Windows OOBE. Each manufacturer implements their own initial setup sequence — there is no single Android OOBE that applies across all devices. The common thread is: network connection, firmware update, Teams application sign-in. The variations are in the OEM-specific screens, settings, and admin access methods that appear before, between, and after those shared steps. Approach each Android device with the OEM's quick start guide in hand, not with an assumption that it works the same way as the last Android device you configured.

---

## Before First Power-On

**Read the OEM quick start guide.** Unlike MTR Windows (which has a broadly consistent OOBE regardless of OEM hardware), Android MTR devices vary significantly in their initial setup sequences. Attempting OOBE without the OEM guide leads to missed steps, incorrectly skipped screens, or accidental factory resets triggered by holding the wrong button combination. Download the guide for the specific device model before installation begins.

**Prepare network connectivity.** Android MTR devices require internet access during OOBE for firmware updates and Teams authentication. Where possible, provide a wired Ethernet connection — use a USB-C to Ethernet adapter (or OEM-provided adapter) if the device does not have a native Ethernet port. Bar-form-factor Android devices (Poly Studio X, Yealink MeetingBar, Logitech Rally Bar) often have only Wi-Fi built in. For Wi-Fi connections, use the 5 GHz band and ensure the device is within strong signal range.

**Have the OEM admin credentials available.** Most Android MTR devices have a default OEM admin PIN or password documented in the setup guide. Have this ready before first boot — you will need it to access admin settings during OOBE.

---

## The Android OOBE Sequence

The general sequence across most Android MTR devices:

**Step 1 — Power on and OEM initialization:** The device displays the OEM boot screen (logo), runs hardware initialization, and then launches the OEM setup wizard. Some devices display a registration or warranty prompt before the setup wizard begins.

**Step 2 — Language and region:** Select language, country/region, and time zone. These settings affect the Teams application display and calendar time interpretation. Set time zone to the deployment location — incorrect time zone causes meeting start times to display incorrectly on the console.

**Step 3 — Network setup:** For Ethernet-capable devices, the wired connection should be detected automatically. For Wi-Fi devices, select the network SSID and enter credentials. After network connection, the device will typically attempt to check for a firmware update immediately.

**Step 4 — Firmware update.** This is the most critical step in the Android OOBE. Most Android MTR devices ship with firmware that is not the current production version. The device downloads and applies the current firmware from the OEM's cloud servers. This process:
- Takes 10–30 minutes on a fast connection
- May require multiple consecutive reboots
- **Must not be interrupted.** A partially applied firmware update can leave the device in an unrecoverable state requiring OEM recovery procedures. Do not remove power, disconnect the network, or attempt to skip the update. Wait until the device has fully rebooted and returned to the setup wizard before proceeding.

**Step 5 — Teams application sign-in.** After firmware update, the Teams application either launches automatically or must be launched from the Android app launcher (device-dependent). The sign-in process is functionally identical to MTR Windows: enter the resource account UPN and password, authenticate via Entra ID, and the Teams home screen appears on success.

**Step 6 — OEM admin setup.** During or after OOBE, the OEM admin menu is accessible via a PIN or button combination (documented in the OEM guide). Access the admin menu and change the default admin PIN to a secure, documented value. Verify network settings (confirm the IP address, check NTP server configuration).

## Key Difference from MTR Windows OOBE

The most important distinction from the Windows path: **Android OOBE firmware updates are mandatory and cannot be deferred.** On MTR Windows, the Teams app update happens in the background and is less disruptive. On Android, the firmware update is part of the OOBE sequence itself and the device will not proceed to Teams sign-in until the update is complete. Plan installation time accordingly — on a new Android device arriving from the warehouse, budget 30–45 minutes for OOBE before the Teams sign-in step.

---

## Post-OOBE Verification

After sign-in, run the same verification sequence as MTR Windows:

- Resource account name appears correctly on the home screen
- Calendar panel shows pre-scheduled test meetings
- Integrated camera feed visible in device settings
- Audio test confirms integrated microphone and speaker function
- Device appears in TAC Devices within 20 minutes with Healthy status

For Android devices with integrated hardware (camera, microphone, speaker built into the bar), peripheral health in TAC is typically reported at the device level rather than as separate peripheral categories. A healthy device status means all integrated hardware is functioning.

## Summary

By completing this lesson, you can now:

- **Prepare for Android OOBE:** OEM quick start guide in hand, network ready, OEM admin credentials available before first power-on.
- **Navigate the OOBE sequence:** OEM boot → language/region → network → firmware update (wait, never interrupt) → Teams sign-in → OEM admin setup and PIN change.
- **Handle firmware update correctly:** Budget 30–45 minutes; never interrupt; wait for explicit error before taking action on a stalled progress indicator.
- **Run post-OOBE verification:** Same checklist structure as Windows — home screen, calendar, camera/audio, TAC status.
