## Overview

Module 3.1 covers the Windows MTR commissioning sequence from first power-on to signed acceptance report. The sequence is methodical: complete OOBE, sign in the resource account, configure the application settings, harden the Windows OS settings, then execute the acceptance test. Each step builds on the previous one — an OOBE completed without network connectivity will require repeating; an acceptance test conducted before the OS settings are correct will pass tests that will fail in production.

This module teaches the sequence, the settings, and the verification approach that produces a room the client can depend on.

---

## What you will learn

By completing this module, you will be able to:

- Navigate the MTR Windows OOBE from first boot through the Teams welcome screen, handling the most common first-boot failure conditions
- Sign an MTR device into a resource account, verify sign-in success across local and TAC indicators, and diagnose sign-in failures from error messages and application logs
- Configure display, audio, meeting, and peripheral settings in the MTR application for a given room specification
- Set Windows power, update, and network settings that prevent reliability issues in production
- Execute all hardware, software, and functional acceptance test items and produce a documentation package for client signoff
