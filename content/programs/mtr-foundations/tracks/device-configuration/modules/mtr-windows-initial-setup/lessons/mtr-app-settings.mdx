## Overview

The MTR application settings are the control layer between the physical hardware and the Teams meeting experience. Misconfigurations here — wrong default microphone, display count set wrong, admin PIN unchanged — generate support calls on the first day of use. Correct settings configuration is part of the commissioning deliverable: the room should not require any settings changes after client acceptance. This lesson covers every settings category, what each option does, and how to configure it correctly for a given room.

---

## Accessing Settings

App settings are accessed by tapping the gear icon (⚙) on the MTR home screen. The app will prompt for the admin PIN. The default PIN on most MTR Windows builds is **0000** — this must be changed to a secure, documented PIN during initial setup. A room whose admin PIN has not been changed from default is accessible by any person who knows the default PIN.

After entering the PIN, the settings menu displays four main categories: **Meetings**, **Display**, **Peripherals**, and **General** (which includes account and calling settings, theming, and diagnostics).

---

## Display Settings

**Number of displays:** Set to match the physical display count — 1 for a single-display room, 2 for a dual-display configuration. Setting this incorrectly causes the MTR app layout to be optimized for the wrong display configuration. On a physical dual-display system with the app set to single display, the second display will be unused or show an extended desktop.

**Content duplication:** When dual display is enabled, this controls whether Display 2 mirrors Display 1 (duplication) or shows content independently (extension). In extension mode (the standard for dual-display MTR rooms), Display 1 shows the participant gallery and meeting UI; Display 2 shows content being shared. Duplication shows the same content on both displays — occasionally requested for rooms with an audience at both ends of a long table.

**Display timeout:** The screen-off timer when the room is idle. Set based on the client's preference — typically 5–10 minutes for commercial rooms. A display that never sleeps generates unnecessary wear and heat; a display that sleeps immediately after each meeting creates a poor user experience when the room is occupied between meetings.

**Front Row:** Enables the Front Row layout option (Teams Rooms Pro license required). Front Row displays remote participants in a persistent strip at the bottom of the display, with shared content above. Available as a default layout option or user-selectable during meetings. Requires at least one large display (86"+) or a second display for an effective experience.

---

## Audio Settings

**Default Microphone and Default Speaker:** These dropdowns list all USB audio devices visible to Windows. Select the device that matches the installed microphone system and speaker system. This is the most common audio configuration error in MTR deployments: the Windows audio system may list multiple audio devices (Windows built-in audio, USB audio bar, content bar), and the wrong selection results in a room that has no functional audio from the remote participant's perspective.

To identify the correct device: the device name in the dropdown matches the USB device name as it appears in Windows Device Manager. If uncertain, test each option using the in-app microphone test function — the test confirms which device is actually picking up audio.

**Microsoft Audio Processor:** When enabled, applies Microsoft's AI-based audio enhancement layer (echo cancellation, noise suppression, voice isolation) on top of the hardware's own processing. Available only for devices that support it (verify in the device's compatibility documentation). For certified audio bars that have their own built-in processing, enabling this may create double-processing artifacts — test with and without before finalizing.

**Meeting speaker volume:** The default volume level for meeting audio. Set to an appropriate level for the room — most conference rooms work well with the volume at 60–70% of maximum to leave headroom for users to adjust up or down during meetings.

---

## Meeting Settings

**Automatic screen sharing:** When enabled, the room automatically shares HDMI-connected content when a device is connected. When disabled, a user must manually start content sharing from the touch console. Client preference varies — some prefer automatic sharing (easier for presenters), others prefer manual (prevents accidental sharing during setup). Confirm the client's preference.

**Hide meeting names and organizer information:** Privacy mode. When enabled, meeting titles and organizer names are not displayed on the room's home screen or touch console. Appropriate for executive suites and rooms in open-plan areas where meeting names might contain sensitive information. The room still joins meetings correctly — the information is only hidden from the home screen display.

**Incoming meeting audio default (unmute/mute):** Sets whether the room's microphone starts unmuted or muted when joining a meeting. For most conference rooms, unmuted is the correct default — the room should be able to speak in meetings without a manual unmute step. For monitoring rooms or rooms primarily used for watching broadcast content, muted default may be appropriate.

**Layout defaults:** Set the default meeting layout (Gallery, Large Gallery, Together Mode, Front Row). Gallery is the standard default for most conference rooms. Together Mode and Front Row are optional experiences that some clients request as the room default.

---

## Peripherals Settings

**Camera assignment:** Select the primary camera from the USB device list. For systems with both a primary meeting camera and a content camera (whiteboard camera), assign each correctly. Misassigning the content camera as the primary camera will send the whiteboard image to remote participants during the meeting instead of the room's participant camera.

**Peripheral health status:** Each peripheral (camera, microphone, speaker) shows a health indicator. A device showing as unhealthy at this stage — with the USB connection verified and the device powered — indicates a USB enumeration problem or driver issue that must be resolved before acceptance.

---

## General Settings

**Room name:** The name displayed on the home screen. Populated from the resource account's Entra ID display name but can be overridden here or in SkypeSettings.xml. Set to the room's common name as the client intends to identify it (e.g., "Board Room A", "Innovation Lab").

**Custom theming:** Upload a custom background image for organizational branding. Requires a JPEG image file accessible to the device (local file or network path). Image specifications: 3840×1080 pixels for single 4K display. Follow the client's brand guidelines for theming — or leave at the Microsoft default if no custom theme is requested.

**Send logs:** Confirm this is enabled. Allows Microsoft Support to request diagnostic log collection when support cases are opened through TAC. Disabling this makes escalated troubleshooting significantly harder.

## Summary

By completing this lesson, you can now:

- **Configure display settings:** Display count, content duplication vs. extension, timeout, Front Row.
- **Assign audio devices correctly:** Default Microphone and Default Speaker matched to installed hardware; verified with in-app test.
- **Set meeting behavior:** Automatic content sharing, privacy mode, incoming audio default, layout defaults — all confirmed with the client before commissioning.
- **Change the admin PIN:** Required at every commissioning; document the new PIN in project records.
