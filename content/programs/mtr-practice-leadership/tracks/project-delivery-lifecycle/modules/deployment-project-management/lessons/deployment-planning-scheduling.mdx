## Deployment Planning Framework

A multi-room MTR deployment is a logistical coordination exercise as much as a technical one. The schedule must account for hardware delivery, network readiness, resource account provisioning, physical installation, and commissioning — across multiple rooms, floors, or sites, often while the building is occupied.

The critical path is the sequence of dependent tasks that determines the minimum project duration. Any delay on the critical path delays the entire project. Tasks off the critical path have float — they can slip within limits without affecting the end date.

---

## Work Package Structure

Break the deployment into discrete work packages. For a standard MTR project:

| Work Package | Description | Typical Duration | Critical Path? |
|---|---|---|---|
| WP1: Hardware procurement | Order, deliver, goods-in check | 4–6 weeks | Yes (starts earliest) |
| WP2: Network remediation | VLAN, QoS, firewall changes | 2–4 weeks | Yes (client-owned) |
| WP3: Resource account provisioning | M365 accounts, policies, calendar | 1–2 weeks | Yes (client-owned) |
| WP4: Physical installation | Mounting, cabling, power | 1–3 days per room | Yes |
| WP5: Configuration and commissioning | App config, account linking, test call | 1–2 hours per room | Yes |
| WP6: Acceptance testing | Structured test per room | 30–60 min per room | Yes |
| WP7: Documentation | As-builts, config guide, test reports | 2–5 days (parallel) | No |
| WP8: End-user training | Training sessions | 1–2 days | No |

### Critical Path Example (20-room single-site project)

```
Week 1-2:   Site survey and NRA complete; hardware order placed; SOW signed
Week 2-4:   Network remediation (client IT; WP2) — concurrent with hardware delivery
Week 3-5:   Hardware delivery and goods-in check (WP1)
Week 4-5:   Resource account provisioning (client IT; WP3)
Week 5:     Pre-installation validation (network + accounts confirmed)
Week 5-7:   Physical installation (WP4) — 4 rooms/day with 2-person team
Week 7:     Commissioning (WP5) — 20 rooms × 1.5 hours = 3 days
Week 7-8:   Acceptance testing (WP6) — with client representative
Week 8:     Documentation and training (WP7, WP8)
Week 8:     Project closeout
```

Hardware procurement and client network/account readiness are always on the critical path. These are the tasks that most commonly slip.

## Scheduling Strategies

### Installation Sequencing

When scheduling physical installation across multiple rooms:

**Floor-by-floor sequencing**: Complete all rooms on one floor before moving to the next. Benefits: fewer elevator trips, single cable management access point, efficient tool transport. Best for: uniform buildings with similar room types per floor.

**Room-type sequencing**: Complete all boardrooms first, then standard meeting rooms, then huddle spaces. Benefits: refine the installation process on the most complex rooms first; less context-switching. Best for: projects with distinct room types requiring different hardware configurations.

**Pilot-first sequencing**: Install and commission 1–3 pilot rooms fully before proceeding to the full fleet. Benefits: validates design, catches unexpected issues before they affect all rooms, gives client early access to live demonstration. Best for: first-time client, novel design, or high-stakes project.

<Callout type="tip">
**Always recommend a pilot phase for projects of 10+ rooms.** The cost of installing one room incorrectly across 10 rooms (and then retrofitting) far exceeds the cost of a 1-week pilot validation period.
</Callout>

### Resource Allocation

For a standard 20-room single-site project:
- **Installation team**: 2 engineers per team; typically 3–4 rooms per day
- **Commissioning**: 1 engineer; 4–6 rooms per day (after physical install)
- **Project manager**: On-site for Day 1 and commissioning; remote coordination otherwise
- **Client IT liaison**: Available for account issues during commissioning week

For multi-site projects, calculate travel time between sites as non-productive time and build it into the schedule.

### Minimising Client Disruption

- Schedule installation during low-occupancy periods (early morning, end of day, or nights/weekends with client approval)
- Phase installation to ensure a percentage of meeting rooms remain available throughout the project
- Provide advance notice (minimum 5 business days) to room administrators before each room goes offline
- Build a 10% schedule contingency for rooms that cannot be accessed as planned

## Schedule Dependencies and Constraints

The dependencies that most commonly create schedule risk:

### Client-Owned Dependencies

These are outside the contractor's direct control:

| Dependency | Lead Time Required | What to Do If It Slips |
|---|---|---|
| Network VLAN/QoS configured | 2 weeks before install week | Escalate to project sponsor; flag as critical path risk |
| Resource accounts provisioned | 5 business days before commissioning | Cannot commission without accounts; postpone commissioning |
| Rooms available for installation | 48 hours confirmation | Re-sequence to available rooms; defer blocked rooms |
| Client IT available for commissioning support | Agreed dates | Book calendar slots at kickoff; confirm week before |

### Contractor-Owned Dependencies

| Dependency | Lead Time Required | Risk Mitigation |
|---|---|---|
| Hardware delivered and checked | 1 week before install | Early delivery buffer; goods-in inspection procedure |
| Installation team confirmed | 3 weeks before install | Book engineers at schedule finalisation; confirm 1 week before |
| Configuration templates ready | 1 week before commissioning | Template preparation as part of pre-install checklist |

## Communication Cadence During Deployment

Proactive communication prevents surprises. Establish these touchpoints:

| Communication | Frequency | Audience | Content |
|---|---|---|---|
| Project status report | Weekly | Project sponsor, IT lead | Progress vs plan, risks, actions |
| Daily site briefing | Daily (installation weeks) | Site team + client IT | Today's rooms, access requirements, blockers |
| End-of-day report | Daily (installation weeks) | Project manager + client IT lead | Rooms completed, issues, tomorrow's plan |
| Milestone notifications | At each milestone | Project sponsor | Go/no-go confirmation |
