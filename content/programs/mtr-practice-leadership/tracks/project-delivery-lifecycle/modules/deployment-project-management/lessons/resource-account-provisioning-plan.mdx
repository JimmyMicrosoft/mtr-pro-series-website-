## Resource Accounts as a Critical Path Item

Resource accounts are the M365 identity that links a Teams Rooms device to its calendar and enables meeting join. Without a provisioned, licensed resource account, a Teams Rooms device cannot be commissioned — it will loop on the initial setup screen indefinitely.

Resource account provisioning is almost always a client IT team responsibility. The project manager's job is to ensure the client IT team has everything they need to provision correctly and on time.

The most common commissioning day failure mode is arriving at a room with a device ready to configure and discovering the resource account does not exist, is incorrectly configured, or is not licensed.

---

## Resource Account Requirements

### What Each Room Needs

Every MTR device requires:

1. **A room mailbox** (Exchange Online or hybrid Exchange)
   - Created as a Room type resource account
   - Calendar processing settings configured (AutoAccept enabled)
   - Not a shared mailbox; not a user mailbox

2. **An Entra ID user account** linked to the room mailbox
   - Sign-in enabled
   - Password set to never expire
   - No MFA enforced (MTR devices are non-interactive; MFA breaks sign-in)

3. **A Teams Rooms license assigned**
   - Teams Rooms Basic (free, limited features) or
   - Teams Rooms Pro (paid, full features including Pro Management)

4. **A Teams Meeting Room policy assigned**
   - Typically a meeting policy that permits recording, transcription, and content sharing per client governance requirements

5. **Optional: Teams Phone license** (if PSTN calling required)

### Naming Convention

Establish and agree a naming convention before provisioning begins. Consistent naming makes fleet management significantly easier.

Recommended pattern: `[SitePrefixCode]-[FloorCode]-[RoomName]@[tenant].com`

Examples:
- `LON-F2-Boardroom@contoso.com`
- `MAN-F1-MR001@contoso.com`
- `NYC-F3-HuddleA@contoso.com`

Document the agreed naming convention in the Resource Account Provisioning Guide so the client IT team applies it consistently.

## The Provisioning Guide

Provide the client IT team with a written provisioning guide tailored to their environment (cloud-only vs hybrid Exchange). The guide should be specific — not a generic Microsoft tutorial, but a step-by-step walkthrough for their tenant.

### Cloud-Only Provisioning Steps

```powershell
# Step 1: Create the Room Mailbox
New-Mailbox -Name "LON-F2-Boardroom" `
            -Alias "LON-F2-Boardroom" `
            -Room `
            -EnableRoomMailboxAccount $true `
            -MicrosoftOnlineServicesID "LON-F2-Boardroom@contoso.com" `
            -RoomMailboxPassword (ConvertTo-SecureString -String "P@ssw0rd123!" -AsPlainText -Force)

# Step 2: Configure Calendar Processing
Set-CalendarProcessing -Identity "LON-F2-Boardroom" `
                       -AutomateProcessing AutoAccept `
                       -AddOrganizerToSubject $false `
                       -DeleteComments $false `
                       -DeleteSubject $false `
                       -RemovePrivateProperty $false `
                       -AddAdditionalResponse $true `
                       -AdditionalResponse "This room is equipped with Microsoft Teams Rooms."

# Step 3: Disable password expiry
Set-MsolUser -UserPrincipalName "LON-F2-Boardroom@contoso.com" -PasswordNeverExpires $true

# Step 4: Assign license (Teams Rooms Pro)
# Do this in M365 Admin Centre or via Graph API
```

<Callout type="warning">
**Password expiry on resource accounts breaks device sign-in silently.** If a resource account password expires, the MTR device will show "Signing in..." indefinitely with no clear error message. Always configure PasswordNeverExpires and document this requirement clearly.
</Callout>

### Hybrid Exchange Considerations

If the client uses hybrid Exchange (on-premises Exchange + Exchange Online):

- Room mailboxes may need to be created in on-premises Exchange and synced to Entra via Entra Connect
- The Entra ID account must be enabled for cloud sign-in (not just on-premises)
- Licensing must be applied in M365 Admin Centre after the account appears in Entra ID
- The provisioning process is typically 24–48 hours end-to-end due to sync cycles

Document the client's hybrid configuration and adapt the provisioning guide accordingly.

## The Resource Account Tracker

The tracker is the provisioning status document for the deployment. Update it daily during the provisioning window.

```
Room ID | Display Name          | UPN                              | Mailbox Created | License Assigned | Calendar Config | Password Never Expires | Status
--------|----------------------|----------------------------------|-----------------|------------------|-----------------|------------------------|--------
R-001   | LON-F2-Boardroom      | LON-F2-Boardroom@contoso.com     | ✓               | ✓ Pro            | ✓               | ✓                      | Ready
R-002   | LON-F2-MR-2A          | LON-F2-MR2A@contoso.com         | ✓               | ✓ Pro            | ✗               | ✓                      | Pending - calendar config
R-003   | LON-F2-MR-2B          | LON-F2-MR2B@contoso.com         | ✗               | —                | —               | —                      | Not started
```

Share this tracker with the client IT team and the commissioning engineer. Review status 5 business days before the commissioning date — any room not in "Ready" status by this date needs an escalation path.

## Validation Before Commissioning

Before the commissioning engineer arrives at a room, verify:

```powershell
# Check mailbox exists and is a Room type
Get-Mailbox -Identity "LON-F2-Boardroom@contoso.com" | Select DisplayName, RecipientTypeDetails

# Check calendar processing
Get-CalendarProcessing -Identity "LON-F2-Boardroom@contoso.com" | Select AutomateProcessing, AddOrganizerToSubject

# Check license in Teams
Get-CsOnlineUser -Identity "LON-F2-Boardroom@contoso.com" | Select DisplayName, TeamsUpgradeEffectiveMode, AssignedPlan

# Test sign-in (use the Teams Rooms app sign-in test or a browser test)
# Navigate to https://login.microsoftonline.com and sign in as the resource account
# Confirm sign-in succeeds without MFA prompt
```

Run this validation script for every room in the fleet before commissioning week. Address any failures before the commissioning engineer is on site.

## Communicating with the Client IT Team

The provisioning plan communication sequence:

| Milestone | Action | Timing |
|---|---|---|
| SOW signed | Send provisioning guide and tracker template to IT lead | Day 1 |
| Provisioning guide received | Agree provisioning start date and IT resource availability | Week 1 |
| 3 weeks before commissioning | Confirm IT team has started provisioning; review tracker | Week 3 |
| 1 week before commissioning | Run validation script; flag any incomplete accounts | Week 7 |
| 5 days before commissioning | Final tracker review; escalate any remaining gaps to IT sponsor | Week 7 |
| Commissioning day | Commissioning engineer confirms account sign-in before physical install | Day of |
