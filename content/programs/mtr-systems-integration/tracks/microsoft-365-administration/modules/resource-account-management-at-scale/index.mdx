## Overview

A single MTR deployment might have 10 resource accounts. An enterprise deployment might have 400. The operational difference is not just scale — it is the difference between doing things manually and doing them systematically. This module covers the PowerShell-based provisioning and management approach that makes large resource account estates maintainable, the license strategy that balances functionality against cost, and the Microsoft 365 policy framework that keeps device accounts secure and compliant.

---

## What you will learn

By completing this module, you will be able to:

- Write and execute PowerShell scripts that provision resource accounts in bulk with all required Exchange Online and Entra ID configuration
- Select the appropriate license tier (Teams Rooms Basic vs. Pro) for each room and manage license assignments efficiently
- Design a Conditional Access policy structure that applies MFA and compliance controls to user accounts while correctly excluding MTR device accounts
- Configure Exchange Online AutomateProcessing, calendar permissions, and room mailbox settings for any number of rooms from a single script
- Establish naming conventions and Entra ID group structures that make the room account estate auditable and self-documenting
