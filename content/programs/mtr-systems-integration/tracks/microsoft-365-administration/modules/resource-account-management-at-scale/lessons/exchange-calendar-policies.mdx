## Overview

Calendar processing configuration is where the room booking behavior is defined. Beyond the basic `AutomateProcessing AutoAccept` covered in Program 1, `Set-CalendarProcessing` has a range of parameters that control booking windows, conflict handling, privacy, and working hours. Enterprise deployments often have specific requirements — some organizations allow only managers or admins to book certain rooms, others restrict booking to business hours only. This lesson covers the full parameter set and the configurations for common enterprise scenarios.

---

## Key Set-CalendarProcessing Parameters

The complete parameter set for enterprise room configuration:

**Booking acceptance:**
```powershell
Set-CalendarProcessing -Identity $room -AutomateProcessing AutoAccept
```
- `AutoAccept` — automatic booking confirmation (standard for self-service rooms)
- `AutoUpdate` — processes updates and cancellations but requires manual booking approval
- `None` — no automatic processing (requires a human to approve all bookings)

**Meeting privacy:**
```powershell
Set-CalendarProcessing -Identity $room `
    -DeleteComments $false `
    -DeleteSubject $false `
    -AddOrganizerToSubject $false `
    -RemovePrivateProperty $false
```
- `DeleteComments $false` — preserves meeting body/agenda text in the room's calendar entry
- `DeleteSubject $false` — preserves meeting subject in the room calendar
- `AddOrganizerToSubject $false` — does not append "Organizer: [Name]" to the meeting subject
- `RemovePrivateProperty $false` — preserves private meeting flags (prevents Teams Rooms from showing meeting details for privacy-flagged meetings)

**Booking window:**
```powershell
Set-CalendarProcessing -Identity $room `
    -BookingWindowInDays 365 `
    -MaximumDurationInMinutes 480
```
- `BookingWindowInDays` — how far in advance rooms can be booked (default: 180 days)
- `MaximumDurationInMinutes` — maximum meeting duration the room will accept (default: 1440 minutes = 24 hours; 480 = 8 hours is more typical for conference rooms)

**Business hours restriction:**
```powershell
Set-CalendarProcessing -Identity $room `
    -ScheduleOnlyDuringWorkHours $true
```
- `ScheduleOnlyDuringWorkHours $true` — the room will only accept bookings that fall within the mailbox's configured working hours
- Working hours are set separately: `Set-MailboxCalendarConfiguration -Identity $room -WorkingHoursStartTime 08:00 -WorkingHoursEndTime 18:00`

**Conflict handling:**
```powershell
Set-CalendarProcessing -Identity $room `
    -AllowConflicts $false `
    -ConflictPercentageAllowed 0 `
    -MaximumConflictInstances 0
```
- `AllowConflicts $false` — rejects meetings that conflict with existing bookings
- `AllowConflicts $true` combined with `ConflictPercentageAllowed` — useful for shared resource rooms where limited double-booking is acceptable

**External meeting requests:**
```powershell
Set-CalendarProcessing -Identity $room -ProcessExternalMeetingMessages $true
```
- `$true` — allows external senders (from federated organizations) to book the room. Required for Teams federation scenarios and Direct Guest Join use cases.

## Restricted Room Configuration

For executive boardrooms or rooms that require administrative approval or can only be booked by specific users:

```powershell
# Option 1: Allow only specific users to book (all-hands approval mode)
Set-CalendarProcessing -Identity $room -AutomateProcessing AutoUpdate
Set-MailboxFolderPermission -Identity "${room}:\Calendar" -User "ea-team@contoso.com" -AccessRights Editor
```

`AutoUpdate` means booking requests are sent to the room delegate for approval — the delegate accepts or declines. Combined with `Editor` permissions for the EA team, only the EA team's acceptances count.

```powershell
# Option 2: Allow only members of a booking group to book directly
Set-CalendarProcessing -Identity $room `
    -AutomateProcessing AutoAccept `
    -AllBookInPolicy $false `
    -BookInPolicy "room-booking-group@contoso.com"
```

`BookInPolicy` specifies a distribution group — only members of this group can submit accepted booking requests. Others receive a decline.

---

## Delegation for Administrative Management

Room delegates can view, accept, or modify room bookings on behalf of the room:

```powershell
# Add a delegate who can manage the room calendar
Add-MailboxFolderPermission -Identity "${room}:\Calendar" `
    -User "ea-admin@contoso.com" `
    -AccessRights Editor

# View current calendar permissions
Get-MailboxFolderPermission -Identity "${room}:\Calendar"
```

Common delegation scenarios:
- Executive assistant managing the boardroom calendar: `Editor` or `PublishingEditor` rights
- Facilities team managing room status: `Reviewer` rights (read-only)
- IT administrator for configuration changes: `Owner` rights

## Batch Calendar Policy Configuration

For large deployments with consistent policy requirements, apply configurations to all rooms from a CSV:

```powershell
$rooms = Get-Mailbox -RecipientTypeDetails RoomMailbox -ResultSize Unlimited

# Apply standard policy to all rooms
foreach ($room in $rooms) {
    Set-CalendarProcessing -Identity $room.UserPrincipalName `
        -AutomateProcessing AutoAccept `
        -DeleteComments $false `
        -DeleteSubject $false `
        -AddOrganizerToSubject $false `
        -RemovePrivateProperty $false `
        -ProcessExternalMeetingMessages $true `
        -AllowConflicts $false `
        -MaximumDurationInMinutes 480 `
        -BookingWindowInDays 365
    Write-Output "Configured: $($room.DisplayName)"
}
```

## Summary

- **Core parameters:** AutomateProcessing, DeleteComments/Subject, RemovePrivateProperty, ProcessExternalMeetingMessages.
- **Booking controls:** BookingWindowInDays, MaximumDurationInMinutes, ScheduleOnlyDuringWorkHours, AllowConflicts.
- **Restricted rooms:** `AutoUpdate` + delegate for approval-required rooms; `BookInPolicy` for group-restricted booking.
- **Delegation:** `Add-MailboxFolderPermission` with appropriate AccessRights for admin management.
