## Overview

Teams Admin Center's device management capabilities scale from a handful of rooms to hundreds — but only if the fleet is organized with tags and filters that make targeted actions possible. Without organization, a 200-device TAC view is a flat list that requires manually identifying individual devices before any action can be taken. With effective tagging, the same 200 devices can be filtered to a building, a room type, or a service tier in seconds, and bulk actions applied to that filtered set.

---

## Navigating to Devices

In Teams Admin Center (`admin.teams.microsoft.com`):
- **Teams Devices → Rooms on Windows** — all MTR Windows devices in the tenant
- **Teams Devices → Rooms on Android** — all MTR Android (bars, boards) devices
- **Teams Devices → Teams panels** — scheduling display panels
- **Teams Devices → Overview** — aggregate health summary across all device types

Each device list shows: device name (resource account display name), device health status, peripheral health, manufacturer, model, app version, and last activity.

---

## Device Tags

Tags are free-text labels applied to devices to support filtering and organization. A device can have multiple tags.

**Applying tags:**
1. Select one or more devices using the checkbox column
2. Click **Actions → Add a tag**
3. Enter a tag name and confirm

**Recommended tag taxonomy:**

| Tag Pattern | Examples | Purpose |
|------------|---------|---------|
| Building code | `bldg:hq`, `bldg:nyc`, `bldg:sfo` | Filter by location |
| Room type | `type:boardroom`, `type:conference`, `type:huddle` | Filter by room category |
| Floor | `floor:01`, `floor:12` | Location precision |
| Service tier | `service:managed`, `service:self-managed` | Managed service scope |
| OEM | `oem:yealink`, `oem:poly`, `oem:logitech` | Platform-specific actions |
| Update ring | `ring:early`, `ring:standard`, `ring:deferred` | Staged update management |

**Adding tags at scale:** TAC does not currently support bulk tag import from a CSV. For large deployments, use the TAC Graph API or PowerShell module (`MicrosoftTeams`) to assign tags programmatically:

```powershell
# Requires MicrosoftTeams module and Teams admin role
Connect-MicrosoftTeams

# Get device IDs for a building and apply a tag
# Note: Device management via Graph API requires specific permissions
# Use TAC UI for most tag management; Graph for automation at scale
```

## Filtering Devices

Use the TAC search and filter capabilities to narrow the device list:

**Search bar:** Searches device name (resource account display name). Type "hq" to find all devices with "hq" in the room name.

**Filter panel (click Filter icon):**
- Health status: Healthy / Non-urgent / Critical / Offline
- Manufacturer: Poly, Yealink, Logitech, Microsoft, etc.
- Model: Filter to specific hardware models
- Tags: Filter by any applied tag
- App version: Filter to find devices on specific app versions

**Combined filters:** Filters are additive — selecting `Health: Non-urgent` AND `Tags: bldg:hq` shows only non-urgent health devices in the HQ building. This is the core workflow for targeted maintenance actions.

---

## Bulk Actions

After filtering to a target device set, select all devices (checkbox at top of list) and choose from **Actions**:

| Action | Description |
|--------|-------------|
| **Restart** | Sends a remote restart command to all selected devices. Devices restart in order — stagger if restarting during business hours |
| **Assign configuration** | Assigns a configuration profile to all selected devices (Pro license required) |
| **Remove configuration** | Removes the assigned configuration profile from all selected devices |
| **Download device logs** | Queues a log collection request from all selected devices |
| **Sign out** | Signs all selected devices out of their resource accounts |
| **Add a tag** | Applies a tag to all selected devices |

**Remote restart workflow:**
1. Filter to target devices (e.g., `bldg:hq` AND `Health: Non-urgent`)
2. Confirm the restart is appropriate (no active meetings in progress)
3. Select all filtered devices
4. Actions → Restart
5. Monitor health status in TAC — devices should return to Healthy within 5–10 minutes

---

## Exporting Device Data

The TAC device list can be exported to CSV for offline analysis or documentation:

1. Filter to the desired device set (or leave unfiltered for full export)
2. Click **Export** (download icon at top right of list)
3. The CSV includes: device name, health status, manufacturer, model, app version, IP address, last activity, tags

The exported CSV is useful for:
- Maintaining an asset register of all deployed rooms
- Identifying devices requiring firmware or app updates (compare app version against current release)
- Feeding into support ticketing or change management workflows

## Summary

- **Navigate:** Teams Devices → Rooms on Windows / Rooms on Android.
- **Tags:** Multi-dimensional organization (building, type, floor, service tier, OEM, update ring) — apply at deployment time.
- **Filter:** Combine health, manufacturer, model, and tag filters to isolate target device sets.
- **Bulk actions:** Restart, assign/remove configuration, download logs, sign out — applied to filtered selections.
- **Export:** CSV export for asset register, version auditing, and change management.
