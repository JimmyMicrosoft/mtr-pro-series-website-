## Overview

Teams Rooms Pro Management is a separate portal from Teams Admin Center, with a distinct URL and purpose. It exists because managing a room estate at scale — particularly under a managed services model — requires more than the basic health indicators TAC provides. Pro Management adds incident management, AI-powered anomaly detection, detailed room-level analytics, and a workflow designed around proactive service delivery rather than reactive break/fix support.

---

## Accessing Pro Management

**URL:** `portal.rooms.microsoft.com`

**Access requirements:**
- The accessing user must have a Teams Rooms Pro license or be a delegated managed service operator
- Access roles within Pro Management:
  - **Managed Services Administrator:** Full access to all portal features
  - **Managed Services Operator:** Operational access (incidents, remote management); no policy configuration
  - **Managed Services Viewer:** Read-only; dashboards and reports only

**Relationship to licensing:** Pro Management is included with the Teams Rooms Pro license. Each room with a Pro license automatically appears in Pro Management. Rooms with Basic licenses do not appear.

---

## Portal Structure

The Pro Management portal is organized into the following main sections:

**Rooms:** The primary view — a list of all enrolled rooms with their current health status, incident count, and last activity. Rooms can be filtered by building, health status, or tags (synchronized from TAC).

**Buildings:** Groups rooms by physical location. Build the building structure in Pro Management to enable building-level health summaries and filtered views. Buildings are configured under Settings → Buildings.

**Incidents:** The incident management feed — all automatically detected issues across the fleet, organized by severity, room, and incident type. Each incident has a status (Open, In Progress, Resolved) and can be assigned to a support team member.

**Tickets:** Integration with external ITSM tools (ServiceNow, Jira) if configured. Pro Management can automatically create tickets in the connected ITSM system when an incident is detected.

**Analytics:** Meeting quality data, room utilization metrics, and trend reports at the room, building, and fleet level.

**Settings:** Portal configuration including building setup, notification rules, delegated access (for managed service providers), and ITSM integration.

## TAC vs. Pro Management

| Capability | TAC | Pro Management |
|------------|-----|----------------|
| Device health status | ✓ Basic (Healthy/Non-urgent/Critical/Offline) | ✓ Detailed with AI analysis |
| Peripheral health | ✓ Per peripheral | ✓ Per peripheral with trend |
| Automated incidents | ✗ | ✓ AI-detected, categorized |
| Remote restart | ✓ | ✓ |
| Log collection | ✓ | ✓ |
| Meeting quality data | ✗ (via CQD separately) | ✓ Room-level quality |
| Utilization analytics | ✗ | ✓ |
| ITSM integration | ✗ | ✓ |
| Multi-tenant management (MSP) | ✗ | ✓ |

Use TAC for: device configuration profiles, update policies, meeting policy assignment, and basic monitoring without Pro licenses.

Use Pro Management for: proactive incident detection, detailed investigation, managed services delivery, utilization reporting, and multi-tenant MSP operations.

## Multi-Tenant Access for Managed Service Providers

For integrators delivering managed services across multiple client tenants, Pro Management supports delegated multi-tenant access:

- The MSP creates a managed service operator account
- Each client tenant grants the MSP operator account access to their Pro Management portal
- The MSP operator sees all managed clients in a unified view at `portal.rooms.microsoft.com`
- Incidents, rooms, and analytics from all managed tenants are accessible from a single portal login

This eliminates the need to maintain separate admin credentials for each client tenant — a single operator login provides cross-tenant visibility appropriate for managed service delivery.

## Summary

- **Access:** `portal.rooms.microsoft.com`; Pro license required per room; Managed Services roles for granular access control.
- **Structure:** Rooms → Buildings → Incidents → Analytics → Settings.
- **vs. TAC:** Pro Management adds AI incident detection, meeting quality data, utilization analytics, ITSM integration, and multi-tenant MSP access.
- **MSP access:** Delegated multi-tenant operator account provides unified view across all managed client tenants.
