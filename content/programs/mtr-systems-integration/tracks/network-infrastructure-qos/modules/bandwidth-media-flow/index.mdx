## Overview

Knowing that the network is correctly configured is different from knowing that it can handle the actual load a deployment will place on it. This module covers the media codecs Teams uses and the bandwidth each requires, how to model aggregate demand for a multi-room deployment, the Call Quality Dashboard tools Microsoft provides for ongoing quality monitoring, and the network readiness validation procedures that confirm the network is meeting production requirements before and after deployment.

---

## What you will learn

By completing this module, you will be able to:

- Identify the Teams media codecs and calculate per-session bandwidth requirements at different quality levels
- Build a capacity model for a multi-room site deployment and validate the WAN/internet connection against peak demand
- Navigate the Call Quality Dashboard, interpret quality metrics, and identify the root cause of quality degradation
- Execute a structured network readiness validation and document the results
