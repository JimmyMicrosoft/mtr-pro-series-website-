## Overview

MTR devices live at the intersection of the AV world and the enterprise network. Getting the network layer right — before devices are deployed — prevents the majority of post-deployment support issues. This module covers the network design decisions that set up an MTR deployment for reliability: VLAN and subnet design, the Microsoft 365 connectivity requirements that define the firewall rule set, QoS implementation that protects meeting media from competing traffic, and Wi-Fi design for rooms that cannot use wired connections.

---

## What you will learn

By completing this module, you will be able to:

- Design a network topology for MTR deployment including dedicated VLANs, IP addressing, and inter-VLAN routing considerations
- Build a complete firewall rule set from Microsoft's published M365 network requirements for Teams Rooms
- Implement DSCP QoS marking on switches and MTR endpoints and verify correct marking in the network path
- Evaluate a Wi-Fi environment against MTR requirements including band, channel width, roaming behavior, and client density
